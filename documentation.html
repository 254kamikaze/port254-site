What is port254?

port254 is a honeypot project designed to study and detect cyber threats targeting critical infrastructure systems. Think of it as a digital trap that looks like a real water treatment plant control system, but is actually monitoring who tries to break in and what they try to do.

Simple Explanation: We've created a fake water treatment plant system that cyber attackers think is real. When they try to hack it, we record everything they do to learn their tactics and improve security for actual critical infrastructure.

Why "port254"?

In industrial systems, "port" refers to communication channels (like doors) that systems use to talk to each other. Common industrial ports include 102 (S7), 502 (Modbus), and others. The "254" represents the Australian country code (+61) and Western Australia's area - this is a Perth-based security research project.

The Problem We're Solving

Critical infrastructure like water treatment plants, power stations, and manufacturing facilities are increasingly connected to the internet. While this connectivity improves efficiency, it also creates opportunities for cyber attackers to cause real-world damage.

How It Works

The Setup

Our honeypot mimics a Perth Water Treatment Plant with several components that look and behave like real industrial control systems:

SSH Gateway

The main entry point - attackers connect here thinking they're accessing the plant's control system.

Port 22

HMI Dashboard

A realistic web interface showing plant operations - pump controls, chemical levels, and system status.

Port 8080

S7 Protocol

Simulates Siemens PLC communication used in real industrial automation.

Port 102

Modbus Protocol

Common SCADA protocol for connecting sensors and controllers.

Port 502

MQTT Broker

IoT messaging system for sensor data exchange.

Port 1883

The Breadcrumb Trail

When attackers connect to our SSH gateway, they see a realistic login banner that mentions the "Perth Water Treatment Plant" and includes a deliberately "leaked" credential for the HMI dashboard. This encourages them to explore further, allowing us to observe their complete attack chain.

Example Attack Flow: Attacker scans for open ports → Finds SSH open → Sees the banner with HMI credentials → Logs in to explore → Visits HMI dashboard → Attempts to modify plant controls → All actions logged for analysis

What We Learn

Every interaction with our honeypot provides valuable intelligence:

  • Attack Patterns: Which services do attackers target first? What commands do they run?
  • Credential Attacks: What username/password combinations are being tried?
  • Protocol Abuse: How are attackers manipulating industrial protocols like Modbus and S7?
  • Geographic Sources: Where in the world are attacks originating?
  • Tool Fingerprints: What hacking tools and scripts are being deployed?

Detection Development

The data collected feeds into our detection catalog - a library of security rules that can identify these attack patterns in real industrial systems. Each detection is mapped to:

  • MITRE ATT&CK: Industry-standard framework for categorizing cyber threats
  • IEC 62443: International security standards for industrial automation
  • Real Telemetry: Actual data from our honeypot showing the detection in action

Safety & Ethics

No Real Systems at Risk

It's important to understand that nothing in this honeypot is real. There is no actual water treatment plant, no real pumps, no chemicals, and no public water supply connected to this system. Everything is simulated in a controlled environment.

Safety First: All honeypot systems are completely isolated from real infrastructure. Attackers cannot cause any real-world harm through our systems.

Responsible Disclosure

When we discover new attack techniques or vulnerabilities through our research, we work with industrial system vendors and security organizations to help protect real critical infrastructure. Our goal is to improve security for everyone, not to expose vulnerabilities maliciously.

Data Privacy

We collect only technical data about attack methods. We don't use this system to target specific individuals, and we share our findings with the security community to help defend against these threats.

Real-World Impact

The intelligence gathered from port254 helps security teams protect actual critical infrastructure by:

  • Providing early warning of new attack techniques targeting industrial systems
  • Creating detection rules that can be deployed in real SCADA environments
  • Understanding attacker motivations and target selection
  • Training security professionals on realistic industrial cyber threats
  • Contributing to open-source security tools and frameworks

Contributing to Security: Our detection catalog is designed to be compatible with popular security platforms like ELK Stack, Splunk, and QRadar - allowing security teams to deploy these defenses in their own environments.

Technology Stack

For those curious about the technical side, port254 is built using:

  • Cowrie: SSH/Telnet honeypot that logs all attacker commands
  • Conpot: Industrial protocol honeypot supporting S7, Modbus, and more
  • Custom HMI: Web-based SCADA interface built with Python and HTML/CSS
  • Docker: Containerization for easy deployment and isolation
  • ELK Stack: Elasticsearch, Logstash, and Kibana for log analysis
  • Geographic Mapping: KML files showing fake plant locations
